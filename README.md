## Overview

Lumora is a full-stack content platform built as a TypeScript monorepo. It provides a clean, secure environment for content creation, publishing, and administration — backed by enterprise-grade security, observability, and containerized deployment.

Perfect for developers who need:
- A production-ready blog and content management system
- Secure authentication with OAuth providers
- Media management with Cloudinary integration
- Admin dashboard with analytics and audit logs
- A monorepo starting point with best practices

---

## Features

### Core Features
- **Authentication** — JWT with `httpOnly` refresh cookies, OAuth (Google/GitHub), email verification, password reset
- **Blog Management** — Full CRUD operations, slugs, tags, categories, draft/published states, SEO metadata
- **Media Library** — Cloudinary uploads with folder organization, multiple media types, metadata support
- **Admin Panel** — User management, analytics dashboard, audit logs, RBAC

### Platform Features
- **Notifications** — In-app notification system with read/unread tracking
- **Global Search** — Search across posts, media, and users with Redis caching
- **Caching** — Redis-backed with graceful degradation when unavailable
- **Observability** — Request logging, error tracking, performance monitoring

### Developer Experience
- **Monorepo Architecture** — Turborepo for efficient builds and caching
- **Shared Packages** — Reusable types, validators, and UI components
- **Type Safety** — End-to-end TypeScript with strict mode enabled
- **Code Quality** — ESLint 9, Prettier, Husky, lint-staged, commitlint

## Installation

### 1. Clone the repository

```bash
git clone https://github.com/yourusername/lumora.git
cd lumora
```

### 2. Install dependencies

```bash
pnpm install
```

### 3. Configure environment variables

```bash
cp .env.example .env
```

Edit `.env` with your configuration:

```env
# Database
DATABASE_URL="postgresql://user:password@localhost:5432/lumora"

# Redis (optional)
REDIS_URL="redis://localhost:6379"

# JWT
JWT_ACCESS_SECRET="your-access-secret"
JWT_REFRESH_SECRET="your-refresh-secret"

# OAuth (optional)
GOOGLE_CLIENT_ID=""
GOOGLE_CLIENT_SECRET=""
GITHUB_CLIENT_ID=""
GITHUB_CLIENT_SECRET=""

# Cloudinary (for media uploads)
CLOUDINARY_CLOUD_NAME=""
CLOUDINARY_API_KEY=""
CLOUDINARY_API_SECRET=""

# Email (Resend)
RESEND_API_KEY=""

# App
NODE_ENV="development"
PORT=4000
FRONTEND_URL="http://localhost:5173"
BACKEND_URL="http://localhost:4000"
```

### 4. Initialize the database

```bash
# Generate Prisma client
pnpm db:generate

# Run migrations
pnpm --filter @lumora/database prisma migrate dev

# Seed database (optional)
pnpm db:seed
```

## Usage

### Development

Start all services in development mode:

```bash

```

This starts:
- **API** — [http://localhost:4000](http://localhost:4000)
- **Web** — [http://localhost:5173](http://localhost:5173)

## Project Structure

```
lumora/
├── apps/
│   ├── api/                    # Express REST API (port 4000)
│   │   ├── src/
│   │   │   ├── modules/        # Feature modules (auth, blog, media, etc.)
│   │   │   ├── middleware/     # Express middleware
│   │   │   ├── utils/          # Utilities (jwt, cache, logger)
│   │   │   └── config/         # Configuration files
│   │   └── test/               # API tests
│   │
│   └── web/                    # React + Vite frontend (port 5173)
│       ├── src/
│       │   ├── pages/          # Route components
│       │   ├── features/       # Feature-specific components
│       │   ├── components/     # Shared components
│       │   ├── hooks/          # Custom React hooks
│       │   ├── store/          # Zustand stores
│       │   └── services/       # API client services
│       └── test/               # Frontend tests
│
├── packages/
│   ├── database/               # Prisma client, schema, migrations
│   ├── shared/                 # Shared types, enums, constants
│   ├── validators/             # Zod schemas (shared across api + web)
│   ├── ui/                     # Radix UI + Tailwind component library
│   └── config/                 # Shared ESLint / TypeScript configs
│
├── docker/                     # Dockerfiles and compose
├── tests/e2e/                  # Playwright E2E tests
└── docs/                       # Documentation
```

**Build order:** `database` → `shared` → `validators` → `ui` → `api` + `web`

## API Reference

All routes are prefixed with `/api/v1`. Interactive documentation available at `/api-docs` when running.

| Module        | Path                | Description                              |
| ------------- | ------------------- | ---------------------------------------- |
| Auth          | `/api/v1/auth`      | Register, login, logout, OAuth, password reset |
| Blog          | `/api/v1/blog`      | Post CRUD, tags, categories              |
| Media         | `/api/v1/media`     | Upload, list, delete files               |
| Admin         | `/api/v1/admin`     | Users, analytics, audit logs             |
| User          | `/api/v1/user`      | Profile, notifications                   |
| Search        | `/api/v1/search`    | Global search                            |

### Authentication

The API uses a dual-token system:
- **Access Token** — Short-lived (15 min), sent as `Bearer` token
- **Refresh Token** — Long-lived (7 days), stored in `httpOnly` cookie
